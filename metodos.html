¿Qué son los métodos?

Los métodos son bloques de código que realizan una tarea específica dentro de un programa. Su principal objetivo es organizar mejor el código y evitar repetir instrucciones innecesariamente.

Importante: Los métodos permiten dividir un programa en tareas más pequeñas, haciendo que el código sea más fácil de entender, reutilizar y mantener.

¿Para qué sirven?

  • Organizar mejor los programas.
  • Reutilizar código en diferentes partes del programa.
  • Reducir la repetición de instrucciones.
  • Facilitar el mantenimiento y la corrección de errores.
  • Hacer que el código sea más legible y profesional.

Aplicaciones reales

  • Calcular promedios de estudiantes.
  • Validar datos ingresados por el usuario.
  • Procesar ventas y facturas.
  • Realizar operaciones matemáticas repetitivas.
  • Organizar programas grandes en partes más pequeñas.

Estructura de un método

La estructura general de un método en Java es la siguiente: 


// Declaración de un método
public static tipoRetorno nombreMetodo(parametros) {
    // Instrucciones que realiza el método
}

Cada parte cumple una función:

  • public: Indica que el método puede ser utilizado desde otras clases.
  • static: Permite llamar al método sin crear un objeto de la clase.
  • tipoRetorno: Es el tipo de dato que devuelve el método (int, double, String, void).
  • nombreMetodo: Nombre que identifica al método. Usa camelCase.
  • parámetros: Datos que recibe el método para trabajar (pueden ser varios o ninguno).

Tipos de métodos

Tipo Descripción Palabra clave
Con retorno Devuelve un valor usando return. int, double, String, etc.
Sin retorno Realiza una acción pero no devuelve nada. void
Con parámetros Recibe datos para procesar. (tipo nombre, tipo nombre)
Sin parámetros No necesita datos externos. ()

Ejemplo básico: método sin retorno y sin parámetros

El siguiente método simplemente muestra un mensaje en pantalla. No recibe datos ni devuelve nada: 


// Método que muestra un saludo en pantalla
public static void saludar() {
    // Imprime el mensaje de bienvenida
    System.out.println("Bienvenido al programa");
}

Para ejecutar el método se debe llamar desde el programa principal: 


// Llamada al método desde el main
saludar();  // Muestra: Bienvenido al programa

Métodos con retorno

Un método con retorno devuelve un valor mediante la palabra reservada return. El tipo de dato del retorno debe coincidir con el declarado. 


// Método que recibe dos enteros y devuelve su suma
public static int sumar(int a, int b) {
    // Retorna la suma de a y b
    return a + b;
}

En este ejemplo el método recibe dos números enteros y devuelve su suma.

Métodos sin retorno

Cuando un método no devuelve ningún valor se utiliza la palabra void. Estos métodos generalmente realizan una acción como imprimir en pantalla. 


// Método que muestra un saludo simple
public static void saludar() {
    // Imprime el mensaje
    System.out.println("Hola");
}

Ejemplo teórico: método con retorno explicado

El siguiente programa utiliza un método para sumar dos números. Es un ejemplo sencillo para entender el concepto: 


// Clase principal
public class Metodos {

    // Método que suma dos números enteros y devuelve el resultado
    public static int sumar(int a, int b) {
        // Retorna la suma de los dos parámetros
        return a + b;
    }

    // Método principal (punto de entrada del programa)
    public static void main(String[] args) {

        // Llamamos al método sumar con los valores 10 y 5
        int resultado = sumar(10, 5);

        // Mostramos el resultado en pantalla
        System.out.println("La suma es: " + resultado);  // Imprime: La suma es: 15
    }
}

Análisis línea por línea

  • public → permite acceder al método desde otras clases.
  • static → permite llamar al método sin crear un objeto.
  • int → indica que el método devuelve un número entero.
  • sumar → es el nombre del método.
  • (int a, int b) → son los parámetros que recibe el método.
  • return → devuelve el resultado de la operación al main.

Explicación del código

  • Se crea el método sumar() fuera del main().
  • El método recibe dos parámetros enteros: a y b.
  • La instrucción return devuelve la suma de ambos.
  • Desde main() se llama al método pasando dos valores.
  • El resultado se almacena en la variable resultado.
  • Finalmente se imprime el resultado en pantalla.

Ventajas de utilizar métodos

  • Evitan escribir el mismo código varias veces.
  • Facilitan la corrección de errores (solo se corrige en un lugar).
  • Permiten reutilizar funciones en diferentes partes del programa.
  • Hacen que el código sea más profesional, modular y organizado.

Métodos con parámetros y sin parámetros

Un método puede recibir datos mediante parámetros o trabajar sin ellos.

Método sin parámetros


// Método que no recibe datos, solo muestra un mensaje fijo
public static void mostrarMensaje() {
    // Imprime un mensaje predefinido
    System.out.println("Hola, bienvenido");
}

Método con parámetros


// Método que recibe un nombre y lo usa para personalizar el saludo
public static void saludar(String nombre) {
    // Imprime el saludo con el nombre recibido
    System.out.println("Hola " + nombre);
}

Método con múltiples parámetros

Un método puede recibir más de un parámetro, separados por comas: 


// Método que recibe nombre y edad, e imprime un mensaje personalizado
public static void mostrarInfo(String nombre, int edad) {
    // Combina texto y número en un solo mensaje
    System.out.println(nombre + " tiene " + edad + " años");
}

Errores comunes

  • Olvidar escribir la palabra return en métodos que deben devolver un valor.
  • Confundir métodos void con métodos que devuelven valores (no se puede usar return con valor en void).
  • Llamar al método con parámetros incorrectos (tipo o cantidad).
  • Escribir mal el nombre del método (Java distingue mayúsculas y minúsculas).
  • Declarar el método dentro de otro método (los métodos deben estar al mismo nivel, uno dentro de la clase).

Ejercicio práctico

Crear un método que reciba dos números enteros y devuelva el número mayor. Posteriormente mostrar el resultado en pantalla.

Solución completa


// Clase principal del ejercicio
public class MetodoMayor {

    // Método que recibe dos números y devuelve el mayor
    public static int mayor(int a, int b) {
        // Si a es mayor que b, devuelve a
        if (a > b) {
            return a;
        } else {
            // Si no, devuelve b
            return b;
        }
    }

    // Método principal (punto de entrada del programa)
    public static void main(String[] args) {

        // Llamamos al método mayor con los valores 15 y 8
        int resultado = mayor(15, 8);

        // Mostramos el resultado en pantalla
        System.out.println("El número mayor es: " + resultado);  // Imprime: El número mayor es: 15
    }
}

Explicación de la solución

  • Se crea el método mayor() que recibe dos enteros.
  • Se comparan los dos números con una estructura if-else.
  • Si el primero es mayor, se devuelve ese valor con return.
  • En caso contrario se devuelve el segundo número.
  • Desde main() se llama al método y se muestra el resultado.

Ejercicio adicional: método con múltiples parámetros y validación

Crear un método que reciba el nombre de un estudiante, tres notas y devuelva su promedio. Además, mostrar si aprobó o no. 


// Clase principal
public class PromedioEstudiante {

    // Método que calcula el promedio de tres notas
    public static double calcularPromedio(double n1, double n2, double n3) {
        // Suma las tres notas y divide entre 3
        return (n1 + n2 + n3) / 3;
    }

    // Método que muestra si el estudiante aprobó o no
    public static void mostrarResultado(String nombre, double promedio) {
        // Imprime el nombre y el promedio obtenido
        System.out.println("Estudiante: " + nombre);
        System.out.println("Promedio: " + promedio);

        // Verifica si aprobó (promedio >= 6.0)
        if (promedio >= 6.0) {
            System.out.println("Estado: Aprobado");
        } else {
            System.out.println("Estado: Reprobado");
        }
    }

    // Método principal
    public static void main(String[] args) {

        // Calculamos el promedio con tres notas
        double promedio = calcularPromedio(7.5, 6.0, 8.0);

        // Mostramos el resultado del estudiante
        mostrarResultado("Ana", promedio);
    }
}

Explicación del ejercicio adicional

  • El método calcularPromedio() recibe tres notas y devuelve el promedio.
  • El método mostrarResultado() es void: recibe el nombre y el promedio, e imprime el estado.
  • Se muestra cómo combinar un método con retorno y uno sin retorno en el mismo programa.
  • Se aplica una validación simple con if-else para determinar si aprobó.

Preguntas frecuentes

¿Qué diferencia existe entre void e int?

void no devuelve ningún valor, mientras que int devuelve un número entero. Otros tipos de retorno pueden ser double, String, boolean, etc.

¿Qué son los parámetros?

Son datos que recibe un método para realizar una tarea específica. Se declaran dentro de los paréntesis del método.

¿Puedo tener dos métodos con el mismo nombre?

Sí, si tienen diferentes parámetros (tipo o cantidad). Esto se llama sobrecarga de métodos.

¿Por qué usar static en los métodos?

static permite llamar al método directamente desde main() sin necesidad de crear un objeto de la clase.

¿Qué pasa si olvido el return?

Si el método declara un tipo de retorno (como int), Java exige que uses return. Si lo omites, el programa no compilará.

Relación con otros temas

Los métodos se combinan con ciclos para realizar tareas repetitivas, con vectores y matrices para procesar colecciones de datos, con listas para recorrer y modificar elementos, y con archivos para leer o escribir datos desde funciones separadas.

Resumen

  • Los métodos organizan el código en bloques reutilizables.
  • Pueden devolver un valor (con tipo de dato) o no devolver nada (void).
  • Pueden recibir parámetros o funcionar sin ellos.
  • Se llaman desde el método main() u otros métodos.
  • Facilitan el mantenimiento, la reutilización y la legibilidad del código.

Conclusión

Los métodos son una de las herramientas más importantes de la programación. Permiten dividir los programas en tareas más pequeñas, reutilizar código y mejorar la organización de las aplicaciones. Su uso es fundamental para el desarrollo de programas más eficientes, modulares y fáciles de mantener.
